Week 3 Notes



The 'this' keyword did take some getting used to for me when I first encountered it in my C++ classes. It's very handy once you understand
what it is. In short, 'this' just refers to the current iteration of a given object.

Declaring objects and accessing the methods and member variables within these objects seems to be similar to C++, although apparently one can
access member variables with brackets and single quotes. That one can't do in C++.
To add a new property into an object, you can use the '.' operator followed by the name of the new property and then set that to whatever value
desired. Use the delete operator to delete a given property from the object. Objects can also be nested within objects.

Note: Objects are copied by reference. The copied object merely points to the original. Changes to either reference will change the other.

To prevent naming collisions, consider using namespaces for potentially repeated generic names of functions or properties.

JSON is a very powerful and incredibly useful tool for building APIs. I've used JSON when getting data from an API call. I know C# has functions
that will parse the data as needed and desired.

Beware of rounding errors with floating numbers. Computers use base 2 math, which is binary. Consequently, it is difficult to have accurate floating
numbers when working with base 2 math. Just make sure your code has some tolerance and doesn't require a high degree of accuracy for floating numbers.

Don't use the getYear() method, since it is not Y2K compliant.

I just learned about RegExp in my database class. I imagine JavaScript and MySQL can work nicely together when using RegExp to call and make queries.

DOM is a wonderful tool to allow modifications to the various parts of an HTML page.

Each element should have a unique ID. Having the same ID for multiple HTML elements will cause odd behavior.

Use 'classList' to modify a particular element's class rather than using 'className'. Using the latter will modify all other classes that have

already been set on the element.

DOM has event listeners that allow you to link JavaScript code to a particular action. For example, if you click on a picture, you can have
some function run to make the picture do something.
Don't use 'click' and 'doubleclick' on the same element. It will likely cause some unexpected behavior because the two listeners may overlap each other.

Don't use anonymous functions as arguments for adding event listeners if you want to eventually remove that listener.
