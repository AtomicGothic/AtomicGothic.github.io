Week 5 Notes



Do your best to program for the benefit of the user. Most users don't have the mind of developers. Even so, programming for users means
making the software experience easy and intuitive for the human mind.

Exceptions are errors that produce a return value.

Follow the call stack to find out where the error happened

Warnings don't mean that the program will stop running, but it may cause issues down the line if not addressed

JavaScript is a language that is pretty forgiving and likes to fail silently in the background. This is not always helpful
To counter this, make sure that your errors will fail to grab your attention. Also use strict mode so that way your JavaScript code will
catch more issues and let you know.

Use alerts and breakpoints to help decode your code. They will interrupt your code at certain points so you can go through the executing
code step by step.

You can throw your own exceptions to make the errors be more readable. Consider using try/catches.

Consider using testing frameworks to help you debug and test your functions
