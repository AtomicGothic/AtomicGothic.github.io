Notate your code. It helps you and your colleagues understand what is going on in the code.

Interestingly, JavaScript doesn't require a semicolon at the end of each statement. JavaScript interpreters use a process called Automatic
Semicolon Insertion to place semicolons where it thinks a semicolon is needed. However, it is best practice to end all of your JavaScript statements
with a semicolon. This comes naturally for me since my fist programming language is C++. Note that blocks do not need to end with a semicolon.

The following are reserved words in the JavScript language:
abstract, await, boolean, break, byte, case, catch, char, class, const, continue, debugger,
default, delete, do, double, else, enum, export, extends, false, final, finally, float, for, function,
goto, if, implements, import, in instanceof, int, interface, let, long, native, new, null, package, private,
protected, public, return, short, static, super, switch, synchronized, this, throw, throws, transient, true,
try, typeof, var, volatile, void, while, with, yield

These words should have been reserved by the language but aren't: undefined, NaN, Infinity.

let and const have block scope, whereas var does not. Keep that in mind.

Interestingly, template literals were introduced to JavaScript. It allows interpolation of JavaScript code. This is similarly done in C#.

Arrays in JavaScript do not have to contain the same type of data. It can contain a mixture of different data types.
"Popping" data off an array does not reduce the size of the array. The empty slots in the array are left as undefined. To remove the slot
completely, you'll need to reduce the size of the array. Alternatively, you can use splice().

Sets seem to be like arrays but not as strict. It is possible to have memory leaks while using sets, so be wary.

Creating maps in JavaScript require the 'new' operator.

The ternary operator is a shorthand way of writing and 'if/else' statement.
JavaScript switch statements are similarly written to C++ switch statements.
Loops in JavaScript have a similar syntax to those in C++. JavaScript does have a 'for of' loop that warrants investigation.

Declaring functions in JavaScript is similar to C++ but you don't have to specify the return type of the function.
Functions can be declared using an "arrow", which looks like this '=>'. It is useful for short, anonymous functions.
Consider using these arrow anonymous functions for callbacks in functions.
