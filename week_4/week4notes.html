Week 4 Notes



Forms are probably the most common way we interact with websites. Websites typically exist to gather information from and give
information to the user. Forms are a way to gather information from the user. One must remember though that all software and websites
exist for the user. Therefore, the interface must be designed for humans. Refer back to CSE 471 for user interface concepts.

Forms are stored in an array in the order they appear on the webpage.

Don't use the reset button to clear out the whole form unless you have a really good reason for it. It's typically considered bad practice.

Alerts are not considered good practice to give user feedback

Normally, submitting a form with the "submit" keyword will send the form data to the back-end to be processed. To interrupt this process,
use preventDefault().

Be sure to indicate what type of input the input element is.

Input types can be checkboxes. Note that you can iterate through the array of forms that are checkboxes to see how many are checked. Useful for
large selection options.

Radio types are like checkboxes but only one option can be selected

There are many input types. Different input types are useful for gathering different kinds of information. Think of it as setting data types
when declaring variables in C++. You want to make sure that you are getting the right kind of data in the right place.

Make sure you validate the data in the forms you create. Validation should occur both on the front end and on the back end. For security reasons,
the back end validation should be the one to rely on because the front end JavaScript validation can be bypassed by savvy users, but JavaScript
validation should still be used to create a better user experience.

I didn't realize that JavaScript could be written in object oriented fashion. Also, it is fortunate that ES6 offers class declaration similar
to other languages like Java and C++, since that is what I'm more familiar with.

Remember, the whole point of OOP is to have code be modular and reusable. Don't have your classes, methods, and functions do more than what they
ought to do. If you need to code new functionality, put it in a different method or maybe a whole new class. Additionally, try to keep coupling minimal
Code that has high coupling runs the risk of spaghettification.

Making class variables private is not done the same way as in C++. Variables can be made private only be limiting scope of the variable. In other words
declare the variable inside the constructor.
